# تحليل موقع ZIP to GitHub Uploader Pro وتحويله إلى بوت

## تحليل الوظائف الأصلية

1. **المصادقة:** إدخال GitHub PAT، إظهار/إخفاء الرمز، التحقق عبر `/user`، وعرض الصورة والاسم واسم الدخول وعدد المستودعات والمتابعين وGists. الرمز في الموقع يبقى في ذاكرة الجلسة.
2. **المستودع:** إنشاء مستودع جديد مع الاسم والوصف والرؤية Public/Private، أو التحقق من مستودع موجود بصيغة owner/name، مع اختيار Branch.
3. **ZIP:** سحب/اختيار ZIP، تحليله في المتصفح بواسطة JSZip، تجاهل المجلدات، ترتيب الملفات، وعرض معاينة حتى 200 ملف.
4. **إعدادات الرفع:** Workers من 1–20، Retry (1/3/5)، Commit message، وBranch.
5. **محرك الرفع:** طابور متوازٍ، قراءة كل ملف Base64، فحص وجوده للحصول على SHA ثم إنشاء/تحديث الملف عبر GitHub Contents API، تأخير تصاعدي، وانتظار Rate Limit.
6. **المراقبة:** ملفات ناجحة/فاشلة/متبقية/إجمالي، الزمن، السرعة، شريط ونسبة تقدم، الملف الحالي، Workers وRetry وRate-limit hits، وسجل طرفية.
7. **الإلغاء والنتائج:** إلغاء، رابط المستودع ونسخه وفتحه، تقرير شامل، Log، وتقرير الملفات الفاشلة، وإعادة بدء عملية جديدة.

## التحويل إلى Telegram

- مراحل الصفحة أصبحت FSM ورسائل Inline: ربط GitHub ← وضع المستودع ← التفاصيل ← Branch ← Commit ← ZIP ← تأكيد ← تقدم ← نتائج.
- بطاقة الحساب أصبحت رسالة بصورة GitHub وCaption منسق.
- Drop Zone أصبح استقبال `Document` من Telegram؛ المعاينة أصبحت عدد الملفات وملخص التأكيد.
- تقدم الصفحة أصبح **تعديل الرسالة نفسها** كل ثانيتين لتجنب Flood Control.
- زر نسخ الرابط يستخدم `copy_text` الحديث، وزر الفتح يستخدم URL مباشر.
- تقارير الموقع أصبحت ملفات `report.txt` و`log.txt` و`failed.txt` مرسلة للمستخدم.

## تحسينات احترافية

- تعدد مستخدمين حقيقي، جلسة FSM منفصلة، وعمليات رفع مستقلة قابلة للإلغاء.
- SQLite بنمط WAL وسجل عمليات دائم وصلاحيات user/admin/blocked.
- تشفير PAT بـ Fernet؛ حذف رسالة الرمز فوراً، وعدم طباعته في السجلات.
- حماية ZIP Slip، حد لعدد الملفات والحجم بعد الفك، واستبعاد `__MACOSX`.
- `aiohttp` Async، Timeouts، Exponential backoff، احترام Retry-After وRateLimit reset.
- تنظيف تلقائي للملفات المفكوكة، Docker، Type hints، واختبارات أساسية.

## قيود واقعية مهمة

- Telegram Bot API السحابي يقيّد تنزيل ملفات البوت عادةً إلى 20MB. لدعم ZIP حتى 2GB شغّل **Local Bot API Server** واضبط البنية وفق متغيرات البيئة؛ الإعداد الحالي يوفّر الحقول اللازمة لكن توصيل خادم محلي يعتمد على عنوانه الفعلي.
- GitHub Contents API لا يقبل ملفاً منفرداً أكبر من 100MB، وكثرة الملفات قد تصطدم بالحدود الثانوية. استخدم Git LFS أو Git Data API/عملية Git محلية للأرشيفات ذات الملفات الضخمة جداً.
- إلغاء طلب HTTP الجاري ليس فورياً؛ يوقف جلب عناصر جديدة من الطابور، ثم تنتهي الطلبات الجارية.
- الموقع الأصلي يدّعي دعم عدة GB داخل المتصفح، لكن ذاكرة JSZip وقيود GitHub تجعل ذلك غير مضمون؛ البوت يفك إلى القرص لتقليل الذاكرة.
